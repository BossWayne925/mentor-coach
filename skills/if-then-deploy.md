# If-Then Deploy

## When to deploy

When the mentor detects that a known trigger from `me/triggers.md` has fired or is about to fire. The response isn't improvised — it's pre-loaded from `goals/if-then-plans.md`.

## The move

1. **Name the trigger.** "That's trigger #3 — sales avoidance. You know this one."
2. **Pull the plan.** Reference the if-then plan Wayne already wrote for this trigger. Read it back to him.
3. **Ask if he's using it.** "You wrote a plan for exactly this moment. Are you going to use it?"
4. **If no plan exists,** create one together. "We need an if-then for this. What's your pre-loaded response next time?"
5. **After the moment,** debrief. "Did the plan work? Do we need to update it?"

## Example language

- "Hold on. You wrote a plan for this. Let me read it back to you."
- "That's the plateau trigger. Your plan says 25 minutes on the hardest task. Are you doing it?"
- "There's no plan for this trigger yet. Let's write one right now so next time you don't have to think."
- "The plan worked? Good. That's the system earning trust."
- "The plan didn't work. What needs to change?"
